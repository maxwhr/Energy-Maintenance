# Energy-Maintenance 软件功能需求分析文档

**项目名称：** Energy-Maintenance  
**系统定位：** 光伏逆变器检修知识检索与作业辅助系统  
**适用范围：** 华为、阳光电源光伏逆变器运维场景  
**文档版本：** v1.0  

---

## 1. 项目概述

Energy-Maintenance 面向光伏电站运维人员、检修工程师和系统管理员，提供设备台账、检修知识库、可追溯问答、故障诊断、SOP 作业辅助、任务管理和记录追溯能力。

第一版系统聚焦华为和阳光电源光伏逆变器，支持设备手册、告警代码、检修规程、故障案例、巡检规范和检修记录的上传、解析、检索和追溯。

---

## 2. 建设目标

系统建设目标如下：

1. 建立光伏逆变器设备台账，统一维护厂家、型号、产品系列、站点和运行状态。
2. 建立检修知识库，支持资料上传、文本解析、知识切片和来源追溯。
3. 提供基于知识库和设备历史的检修问答，输出结构化建议和真实参考来源。
4. 提供故障诊断辅助，根据故障现象、告警信息、设备类型和历史记录生成排查建议。
5. 提供 SOP 作业指引，帮助一线人员按步骤执行检修流程。
6. 建立问答、诊断、任务、模型调用和操作日志的追溯中心。
7. 支持 LoongArch + Kylin 环境下的原生部署。

## 5. 功能需求

### 5.1 用户与权限

系统应支持基础用户管理，包括用户名、姓名、角色、状态和密码哈希字段。第一版至少支持管理员、工程师、操作员等角色，为后续精细权限控制预留基础。

### 5.2 设备台账

系统应支持光伏逆变器设备台账管理，包括设备名称、厂家、产品系列、型号、设备类型、站点名称、安装位置、设备状态和备注信息。

### 5.3 设备检修历史

系统应记录设备历史检修信息，包括关联设备、故障类型、告警码、处理过程、处理结果、负责人、完成时间和附件信息。检修历史应可被问答和诊断模块引用。

### 5.4 知识库管理

系统应支持知识文档上传、解析、清洗、切片和入库。知识文档应包含厂家、产品系列、型号、设备类型、文档类型、来源、解析状态、审核状态和切片数量。

### 5.5 知识审核与纠错

系统应支持知识贡献、知识审核和模型输出纠错。知识管理员可以审核新上传文档或一线人员提交的经验内容，工程师可以对问答或诊断结果进行纠错反馈。

### 5.6 检修问答

系统应根据用户问题、设备类型、厂家、产品系列、型号、告警码和文档类型进行检索，返回结构化回答、建议步骤、置信度、追溯编号和真实来源。

### 5.7 故障诊断

系统应根据故障现象、告警信息、设备状态、图片描述和历史记录，输出可能原因、排查步骤、安全注意事项、推荐处理措施、参考来源和追溯编号。

### 5.8 故障图片与附件

系统应支持故障图片和附件上传，记录文件名、文件路径、扩展名、大小、关联设备、关联诊断记录和图片描述。图片识别和 OCR 能力通过服务抽象预留。

### 5.9 SOP 作业指引

系统应支持 SOP 模板和执行记录。SOP 模板包含适用故障类型、步骤、风险提示和验收要点；执行记录包含执行人员、执行状态、步骤结果和完成时间。

### 5.10 检修任务

系统应支持检修任务创建、查看、状态更新和结果记录。任务可来源于人工创建、问答结果或诊断结果，并通过来源追溯编号关联上下文。

### 5.11 记录追溯

系统应提供问答记录、诊断记录、任务记录、模型调用日志和操作日志查询能力，支持通过 trace_id 追踪一次检修辅助过程。

### 5.12 模型服务

系统通过统一模型服务网关调用本地或云端模型。本地小模型优先采用 llama.cpp + GGUF 路线，云端服务采用 OpenAI-compatible API 接入兼容服务。

### 5.13 系统状态

系统应展示后端服务状态、数据库连接状态、知识库统计、模型服务状态和关键模块运行状态。

## 7. 非功能需求

### 7.1 可用性

系统页面应清晰、专业、适合工业运维场景，避免普通聊天软件风格。

### 7.2 可维护性

后端采用分层架构，前端采用组件化和集中 API 封装，数据库变更通过 Alembic 迁移管理。

### 7.3 可部署性

正式部署目标为 LoongArch + Kylin 原生环境，采用 Python virtual environment、native PostgreSQL、Nginx 和 systemd。

### 7.4 安全性

系统应校验上传文件类型和大小，保存安全文件名，避免路径穿越，并对关键操作保留操作日志。

### 7.5 可追溯性

所有问答、诊断、任务和模型调用均应保留 trace_id，检索回答中的参考来源必须来自真实知识库或业务记录。

---

## 8. 第一版边界

第一版暂不纳入真实大模型生成、embedding、pgvector、OCR 扫描件识别、复杂多租户权限、物联网实时监控、云原生部署和 Kubernetes。上述能力可在核心闭环稳定后逐步扩展。

---

## 9. 验收要点

1. PostgreSQL 能正常连接，迁移能创建所需表。
2. 华为和阳光电源样例资料能上传、解析并生成知识切片。
3. 检修问答能返回真实来源并保存 qa_records。
4. 故障诊断能返回原因、步骤、安全注意事项和处理建议，并保存 diagnosis_records。
5. 检修任务能创建、查询和更新状态。
6. 记录中心能追溯问答、诊断和任务。
7. 前端页面能完成主要业务操作。
8. 系统状态能展示真实数据库和知识库统计。
